## 1. 游戏概述

### 1.1 基本信息

| 项目 | 描述 |
|------|------|
| 类型 | 俯视 2D 自动射击 Roguelite |
| 视角 | 俯视 90° (Top-Down) |
| 操作 | WASD / 摇杆 移动，攻击全自动 |
| 单局时长 | 约 30 分钟（20 波） |
| 目标平台 | PC (键鼠 + 手柄) |
| 画面风格 | 2D 像素 / 扁平矢量（待定） |

### 1.2 核心体验

- **走位是唯一手动操作**——你只能控制移动，攻击完全自动
- **压力递进**——每波敌人越来越多、越来越强
- **Build 构筑**——通过武器和道具组合，配合角色天赋特点产生质变
- **一局一 Build**——每次都是可以选全新的角色 + 道具组合
- **易上手难精通**——操作简单，但走位和 Build 选择决定生死

### 1.3 设计原则

1. **信息透明**：所有数值、效果、公式对玩家可见
2. **即时反馈**：每次升级/购买效果立即可感知
3. **风险回报**：高难度波次 = 更多资源
4. **无陷阱选择**：每个道具/武器在特定 Build 下都有价值
5. **可控随机**：使用加权随机 + 保底机制，避免极端坏运气

## 3. 玩家角色系统

### 3.1 角色属性

每个角色由以下数据定义：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | string | 唯一标识，如 "well_rounded" |
| `name` | string | 显示名称 |
| `description` | string | 角色特色描述 |
| `unlock_condition` | string | 解锁条件（详见 §11.2） |
| `base_stats` | dict | 初始属性值（详见 §4） |
| `starting_weapon` | string | 初始武器 ID |
| `max_hp_modifier` | float | 对初始 HP 的最大值修改倍率 |
| `special_rule` | string | 特殊规则描述（如有） |

### 3.2 初始角色列表（最小可行版本 v1）

| ID | 名称 | 特色 | 初始属性偏差 |
|----|------|------|-------------|
| `well_rounded` | 均衡型 | 无突出特点 | 全部 +0% |
| `brawler` | 格斗家 | 近战专精 | 近战伤害 +50%, 远程伤害 -50%, 攻速 +25% |
| `ranger` | 游侠 | 远程专精 | 远程伤害 +50%, 近战伤害 -50%, 范围 +25% |
| `mage` | 法师 | 元素专精 | 元素伤害 +75%, 近战伤害 -50%, 工程 -50% |
| `tank` | 重装 | 生存专精 | HP +50%, 速度 -25%, 伤害 -25% |
| `lucky` | 幸运儿 | 高幸运 | 幸运 +50, 伤害 -10% |
| `speedy` | 疾风 | 极速 | 速度 +50%, HP -25% |
| `engineer` | 工程师 | 工程专精 | 工程 +75%, 近战 -50%, 远程 -50% |

### 3.3 角色选择流程

1. 主菜单 → "开始游戏"
2. 展示角色选择界面（网格/列表）
3. 未解锁角色显示为灰色 + 解锁条件提示
4. 选择已解锁角色 → 确认 → 进入战斗

## 4. 属性系统

### 4.1 属性清单

游戏中共有 15 个核心属性，属性之间通过加成公式产生复合效果。

| # | 属性名 | 英文 Key | 默认值 | 说明 |
|---|--------|----------|--------|------|
| 1 | 最大生命值 | `max_hp` | 取决于角色 | 基础 10~20 |
| 2 | 生命再生 | `hp_regen` | 0 / 秒 | 每秒恢复 HP 量 |
| 3 | 生命偷取 | `life_steal` | 0% | 造成伤害时回复 HP 的概率（每次判定） |
| 4 | 伤害百分比 | `damage_pct` | 0% | 所有伤害的全局加成 |
| 5 | 近战伤害 | `melee_damage` | 0 | 近战武器的固定伤害加成 |
| 6 | 远程伤害 | `ranged_damage` | 0 | 远程武器的固定伤害加成 |
| 7 | 元素伤害 | `elemental_damage` | 0 | 元素武器的固定伤害加成 |
| 8 | 攻击速度 | `attack_speed` | 0% | 减少武器冷却时间 |
| 9 | 暴击率 | `crit_chance` | 3% | 基础暴击率，触发时伤害 ×2 |
| 10 | 工程学 | `engineering` | 0 | 影响炮塔/地雷等部署物属性 |
| 11 | 攻击范围 | `range` | 0 | 增加远程武器射程 + 近战武器范围 |
| 12 | 护甲 | `armor` | 0 | 每点减少所受伤害（有上限） |
| 13 | 闪避率 | `dodge` | 0% | 完全闪避一次攻击的概率（上限 60%） |
| 14 | 移动速度 | `speed` | 0% | 移动速度加成 |
| 15 | 幸运 | `luck` | 0 | 影响掉落率、商店稀有度、暴击判定等 |
| 16 | 收获 | `harvesting` | 0 | 每波结束时获得的额外材料 |

### 4.2 属性计算规则

#### 4.2.1 属性修改器堆叠

所有属性加成均为**加算叠加**（除特别标注外）：

```
最终属性值 = 基础值 + Σ(固定加成) + 基础值 × Σ(百分比加成)
```

例如：基础伤害 10，近战伤害 +3，伤害% +25%：
```
最终近战伤害 = 10 + 3 + 10 × 0.25 = 15.5
```

#### 4.2.2 属性上下限

| 属性 | 下限 | 上限 | 说明 |
|------|------|------|------|
| 闪避率 | 0% | 60% | 硬上限 |
| 暴击率 | 0% | 100% | 可堆满 |
| 攻击速度 | -80% | 无上限 | 冷却不能低于 0.1 秒 |
| 护甲减伤 | 0% | 90% | 护甲值的减伤效果有上限 |

### 4.3 伤害计算公式

```
最终伤害 = 武器基础伤害 
          × (1 + 伤害%) 
          + 类型伤害固定加成
          
暴击时：最终伤害 × 2.0
```

详见 [§9.2 伤害计算](#92-伤害计算)。

## 5. 武器系统

### 5.1 武器分类

| 类别 | 说明 | 伤害类型 | 示例 |
|------|------|----------|------|
| 近战 | 围绕玩家的范围攻击，有击退 | 受 `melee_damage` 加成 | 剑、拳套、棍棒 |
| 远程 | 发射弹丸，朝最近敌人 | 受 `ranged_damage` 加成 | 手枪、霰弹枪、弓 |
| 元素 | 特殊弹道或范围，附带元素效果 | 受 `elemental_damage` 加成 | 火球、闪电链、冰锥 |
| 工程 | 部署物（炮塔、地雷） | 受 `engineering` 加成 | 炮塔、地雷 |
| 原始 | 简易武器，可投掷 | 混合加成 | 石头、长矛、弹弓 |

### 5.2 武器数据结构

```gdscript
# 武器资源定义（.tres 文件）
{
    "id": "stick",                  # 唯一标识
    "name": "木棍",                  # 显示名称
    "class": "melee",               # 类别：melee/ranged/elemental/engineering/primitive
    "tier": 1,                      # 当前品阶（1~4）
    "base_damage": 8,               # 基础伤害
    "cooldown": 1.5,                # 冷却时间（秒）
    "range": 150,                   # 攻击范围（像素）
    "crit_multiplier": 2.0,         # 暴击倍率
    "pierce": 1,                    # 穿透数（命中敌人后继续飞行）
    "bounce": 0,                    # 弹射数（命中后弹向新目标）
    "projectiles": 1,               # 每次攻击发射的弹丸数
    "knockback": 100,               # 击退力度
    "lifesteal_multiplier": 1.0,    # 生命偷取倍率
    "special_effect": null,         # 特殊效果 ID
    "tags": []                      # 标签（如 "fire", "explosive"）
}
```

### 5.3 武器品阶与合成

| 品阶 | 名称 | 颜色 | 伤害倍率 | 合成条件 |
|------|------|------|----------|----------|
| Tier 1 | 普通 | 灰 | ×1.0 | 初始 |
| Tier 2 | 精良 | 蓝 | ×1.5 | 2 个同名 Tier1 |
| Tier 3 | 稀有 | 紫 | ×2.2 | 2 个同名 Tier2 |
| Tier 4 | 传说 | 红 | ×3.0 | 2 个同名 Tier3 |

**合成规则**：
- 玩家最多携带 **6 个武器**
- 当获得一把已拥有的武器时，自动尝试合成（升级品阶）
- 如果 2 个同名同品阶武器均在槽位中 → 自动合并为下一品阶，空出一个槽位
- 如果槽位已满且无法合成 → 新武器替换旧武器（需玩家确认）
- 合成后保留两把武器中较高的属性修改器

### 5.4 武器攻击逻辑

1. 每个武器独立运行冷却计时器
2. 冷却完毕 → 检查目标（最近敌人 / 随机方向）
3. 根据武器类型执行攻击：
   - **近战**：播放动画，对范围内所有敌人造成伤害
   - **远程**：生成子弹节点，向目标方向飞行
   - **元素**：根据具体武器行为（火球→目标飞行、闪电→连锁、冰锥→扇形散射）
   - **工程**：在玩家位置生成部署物节点
4. 统计 `projectiles` 数量，生成对应子弹数
5. 应用 `pierce` / `bounce` 效果

### 5.5 攻击目标选择

```
目标 = FindNearestEnemy(Player.position, weapon.range)

如果没有敌人在范围内：
  - 远程：随机选择一个方向发射
  - 近战：不攻击（等冷却）
```

## 7. 敌人系统

### 7.1 敌人类型

| 类型 | 行为描述 | HP | 伤害 | 速度 | 特殊 |
|------|----------|-----|------|------|------|
| 基础近战 | 直线追踪玩家 | 低 | 低 | 中 | 无 |
| 快速追踪 | 更快追踪玩家 | 低 | 低 | 高 | 数量多 |
| 冲刺型 | 蓄力后高速冲刺 | 中 | 中 | 低→极高 | 冲刺后短暂硬直 |
| 远程射击 | 保持距离，发射子弹 | 低 | 中 | 低 | 子弹可躲避 |
| 召唤型 | 停留原地，召唤小怪 | 高 | 无 | 静止 | 无限召唤（需优先击杀） |
| 坦克型 | 缓慢移动，高血量 | 极高 | 高 | 极低 | 碰撞体积大 |
| 精英 | 随机增益 Buff + 外观变化 | 高 | 高 | 中 | 掉落更多材料 |
| Boss | 第 20 波最终 Boss | 极高 | 极高 | 中 | 多种攻击模式 |

### 7.2 敌人属性缩放

敌人属性随波次递增：

```
敌人 HP  = 基础 HP  × (1 + 0.15 × (波次 - 1))
敌人伤害 = 基础伤害 × (1 + 0.10 × (波次 - 1))
敌人速度 = 基础速度 × (1 + 0.03 × (波次 - 1))  [上限 +50%]
```

### 7.3 敌人生成规则

- 敌人从地图边缘外生成，向地图中心移动
- 生成位置均匀分布在玩家视野外的 360° 范围内
- 前 5 波不生成精英
- 第 6 波起，每波有概率生成 1~2 个精英（概率 = 波次 × 3%）
- Boss 仅在第 20 波生成（1 个）

### 7.4 敌人 AI 状态机

```
IDLE → CHASE → ATTACK → COOLDOWN → CHASE
         ↑                    │
         └────────────────────┘
```

- **IDLE**：刚生成时的短暂过渡
- **CHASE**：向玩家位置移动（按类型有不同的追踪精度）
- **ATTACK**：到达攻击距离后执行攻击（近战=碰撞伤害，远程=发射子弹，冲刺=加速）
- **COOLDOWN**：攻击后的冷却期

## 8. 波次系统

### 8.1 波次结构

| 波次 | 时长(秒) | 敌人数 | 敌人类型 | 材料倍率 | 备注 |
|------|----------|--------|----------|----------|------|
| 1 | 20 | 15 | 基础近战 | ×1.0 | 热身 |
| 2 | 20 | 20 | 基础近战 | ×1.0 | |
| 3 | 25 | 25 | 基础近战 + 快速追踪 | ×1.1 | |
| 4 | 25 | 30 | 同上 | ×1.1 | |
| 5 | 30 | 35 | 基础 + 快速 + 冲刺 | ×1.2 | 第一个难度跳跃 |
| 6 | 30 | 40 | 同上 + 精英(概率) | ×1.2 | |
| 7 | 35 | 45 | + 远程射击 | ×1.3 | |
| 8 | 35 | 50 | 所有类型 | ×1.3 | |
| 9 | 40 | 55 | 所有类型 + 坦克 | ×1.4 | |
| 10 | 40 | 60 | 所有类型 + 召唤型 | ×1.5 | 中局大关 |
| 11 | 45 | 35 | 怪物强度提升 | ×1.5 | 后半段开始 |
| ... | ... | ... | ... | ... | |
| 19 | 60 | 80 | 全类型高密度 | ×2.0 | Boss 前最后一波 |
| 20 | ∞ (至 Boss 死亡) | 持续生成 + 1 Boss | Boss + 全类型 | ×2.5 | 最终波 |

### 8.2 波次流程

1. **波次开始**：显示 "Wave X" 提示（2 秒动画）
2. **波次进行**：按配置持续生成敌人
3. **波次结束条件**：
   - 计时器归零，且地图上敌人数量 < 阈值（5 个）→ 进入商店
   - 或者所有敌人被消灭 → 立即进入商店
4. 如果计时结束但仍有大量敌人 → 持续生成停止，清理剩余敌人后进入商店

### 8.3 材料掉落

- 击杀敌人掉落材料：基础量 = 1~3（随机）
- 波次倍率 × 幸运倍率
- 精英击杀：×3 材料
- 材料自动吸附范围：基础 80px，受 "收获" 属性 + "捡取范围" 道具影响

## 9. 战斗系统

### 9.1 移动系统

- WASD / 左摇杆 → 8 方向移动均可
- 速度 = 基础速度（200 px/s）× (1 + 速度%)
- 玩家与敌人碰撞 → 受到碰撞伤害（近战敌人接触即造成伤害）
- 玩家不能离开地图边界

### 9.2 伤害计算

```
# 步骤 1：计算攻击的基础伤害
攻击基础伤害 = 武器.基础伤害

# 步骤 2：添加全局伤害加成
加成后伤害 = 攻击基础伤害 × (1 + 伤害%)

# 步骤 3：添加类型伤害加成
加成后伤害 += 武器类型对应的固定伤害加成
# 如：近战武器 += 近战伤害
#      远程武器 += 远程伤害
#      元素武器 += 元素伤害
#      工程武器使用工程学影响部署物属性

# 步骤 4：暴击判定
暴击随机数 = Random(0, 100)
if 暴击随机数 < (基础暴击率 + 暴击率属性):
    加成后伤害 ×= 武器.暴击倍率

# 步骤 5：目标减免
# 护甲减伤（非线性）：
护甲减伤率 = 护甲值 / (护甲值 + 100)
上限 90%

# 闪避判定：
闪避随机数 = Random(0, 100)
if 闪避随机数 < 闪避率:
    伤害 = 0  # 完全闪避

# 步骤 6：最终伤害
最终伤害 = 加成后伤害 × (1 - 护甲减伤率)
```

### 9.3 生命偷取

```
每次造成伤害时：
if Random(0, 100) < 生命偷取率:
    回复 HP = max(1, 伤害 × 0.1)  # 至少回复 1 点
```

### 9.4 死亡条件

- 玩家 HP ≤ 0 → 游戏结束
- 进入结算界面
- 玩家受到伤害时有 0.5 秒**无敌帧**（防止连续被多敌人碰撞秒杀）

## 10. 商店系统

### 10.1 商店展示规则

- 每波结束后进入商店
- 展示 **4 个道具槽** + **0~2 个武器槽**（武器出现概率 = 40% × (1 + 幸运/200)）
- 道具稀有度由幸运加权随机
- 同一种道具不会在同一商店中出现两次

### 10.2 刷新与锁定

- **刷新**：花费材料（价格 = 当前波次 × 1），重置所有槽位
- **锁定**：锁定一个槽位，下次刷新时该槽位不变
- 每波只能刷 1 次，特定道具可增加刷新次数

### 10.3 道具价格计算

```
道具价格 = 基础价格 × (1 + 0.05 × (波次 - 1))
武器价格 = 基础价格（固定，不受波次影响）
```

### 10.4 商店操作

| 操作 | 说明 |
|------|------|
| 购买 | 点击道具/武器，消耗材料，放入玩家库存 |
| 刷新 | 消耗材料，替换所有未锁定槽位 |
| 锁定 | 锁定槽位，刷新/下一波时保留 |
| 继续 | 确认购买完毕，进入下一波 |
